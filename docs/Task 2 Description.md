## **Task Overview** 

Image generative models (IGMs) are trained on data scraped from the internet, much of which was itself produced by earlier generative models. This creates data circuits where an IGM is fine-tuned on generated samples, so the boundary between natural training, test, and generated data becomes blurred. 

Consider a detector that assigns each sample to one of three categories: **M** (member training samples), **N** (non-member samples from the same distribution as the training set), or **G** (model-generated samples). Your goal is to manipulate the detector’s predictions so that it makes the following targeted errors: **M → N, M → G** , **N → M** , **N → G, G → M** , **G → N,** where A **→** B denotes a sample from class A being classified as class B. 

## **What you have access to:** 

- Image generative model (RAR). 

- A dataset of 900 images evenly distributed across three classes (M, N, and G), with 300 images per class. 

## **What you do not have access to:** 

- The detector to classify samples as being **M** , **N** , or **G** . 

## **Evaluation Metric** 

The detector is based on 2 objectives: 

- Membership Inference Attack (MIA) score: detect if an image is a training sample of the given model. 

- Generated image attribution: detect if an image is generated by the given model. 

Your performance is based on: 

- Misclassification by the detector. 

- Image Quality Measure (Mean Squared Error). 

You will submit images for evaluation. If a submitted image induces the correct targeted misclassification direction, it passes the detector check and receives DetectorScore=1; otherwise, it receives the DetectorScore=0. For each passing image, its quality is measured as (1 − MSE), where MSE is the normalized mean squared error between the submitted image and the original reference image (the lower the MSE the better). ## **Submission Format** 

The submission file should be a binary .npz compressed file of your modified images that contains image IDs from 0000 to 1799. 

## **Requirements** 

- Each image must appear exactly once 

- All of the images should be included in the compressed file 

- All of the images must be in the shape of 256 x 256 x 3 

- You should submit the images with the same IDs as the naming convention: 

   - 0000 to 0299 for **M → N** , 

   - 0300 to 0599 for **M → G,** 

   - 0600 to 0899 for **N → M** , 

   - 0900 to 1199 for **N → G,** 

   - 1200 to 1499 for **G → M** , 

   - 1500 to 1799 for **G → N** 

Submissions that do not meet these requirements will be rejected automatically. 

## **Scoring** 

All submissions are automatically validated and evaluated on the hidden test set. 

- Public leaderboard: based on 30% of data 

- Private leaderboard: final ranking based on the remaining 70% 

A higher score indicates more data points that fooled the detector and preserve the quality of the image.
